# C++ TCP Echo Server

## 📡 项目功能概述

本项目是一个基于 C++ 开发的高性能 TCP 回声服务器系统。采用 `epoll` I/O 多路复用技术与固定大小线程池模型，支持高并发客户端连接处理，适用于学习 Linux 网络编程、多线程同步及 TCP 粘包问题解决方案。

客户端发送的消息以换行符 (`\n`) 为分隔符，服务端接收后按协议拆包并原样回显。该设计有效解决了 TCP 字节流传输中的粘包问题，确保消息边界清晰。系统具备良好的可维护性与扩展性，是理解现代网络服务架构的理想范例。

应用场景包括但不限于：
- 学习 socket 编程核心流程（socket → bind → listen → accept）
- 掌握 epoll 事件驱动机制
- 实践线程池与任务队列的并发模型
- 分析应用层协议设计原理

## 🧩 核心组件说明

### `common.h` —— 公共定义与接口

此头文件定义了项目中所有模块共享的常量、数据结构和函数声明，是整个系统的公共契约。

主要包含以下内容：
- **编译期常量**：  
  `MAX_EPOLL_EVENTS`（最大监听事件数）  
  `THREAD_POOL_SIZE`（线程池大小，默认为 **64**）  
  `BUF_READ_SIZE`（读取缓冲区大小，默认为 **1024**）  
  `PORT`（服务监听端口，默认为 **8888**）  
  `LINE_SEP`（协议分隔符，值为 `\n`）

- **共享资源**：  
  `taskQueue`：用于存放待处理客户端连接描述符的任务队列  
  `qunenMutex`：保护任务队列的互斥锁  
  `queueCond`：通知工作线程有新任务到达的条件变量

- **函数声明**：  
  `splitPacketByline()`：根据 `\n` 拆分接收到的数据流  
  `initThreadPool()`：初始化线程池  
  `addTaskToPool()`：将新连接加入任务队列

所有源文件均通过 `#include "common.h"` 引入这些定义，保证接口一致性。

### `server.cpp` —— 服务端实现

该文件实现了完整的 TCP 服务器逻辑，核心流程如下：

1. 调用 `createListtenFd()` 创建监听套接字，并绑定至指定地址与端口。
2. 初始化 epoll 实例，将监听套接字注册为可读事件。
3. 启动主事件循环，使用 `epoll_wait()` 监听新连接请求。
4. 当有新客户端接入时，调用 `accept()` 获取连接套接字，并通过 `addTaskToPool()` 提交至线程池异步处理。
5. 工作线程从任务队列中取出连接，在独立缓冲区中持续接收数据，利用 `splitPacketByline()` 解析出完整消息包，并执行回显操作。

关键特性：
- 使用非阻塞 I/O 配合 epoll ET 模式提升性能
- 每个客户端由独立线程处理，避免阻塞主线程
- 内置日志输出，便于调试与状态追踪

### `client.cpp` —— 客户端实现

客户端提供交互式终端界面，允许用户输入文本并与服务器通信。

主要行为流程：
1. 创建 TCP 套接字并通过 `connect()` 连接到预设 IP 地址（**192.168.230.128**）与端口（**8888**）。
2. 循环提示用户输入消息，自动在末尾追加 `\n` 以符合服务端协议要求。
3. 调用 `send()` 发送数据，随后调用 `recv()` 接收回声响应并打印到控制台。
4. 若输入内容为 `"exit"`，则主动关闭连接并退出程序。

优势特点：
- 用户体验友好，实时反馈交互结果
- 自动协议适配，无需手动添加分隔符
- 错误处理完善，网络中断时能正确退出

> **注意**：若需连接其他主机，请修改 `SERVER_IP` 宏定义后重新编译。

## ⚙️ 项目构建方法

### 前置依赖

为成功构建本项目，请确保开发环境满足以下要求：

- **操作系统**：Linux（需支持 `epoll` 和 POSIX 线程）
- **编译器**：g++ 或 clang++，支持 C++11 及以上标准
- **必需库**：POSIX 线程库（pthread）

推荐使用 Ubuntu/Debian/CentOS 等主流发行版进行开发与测试。

### 编译命令

使用以下 bash 命令完成编译：

```bash
# 编译服务器程序
g++ -o server server.cpp -std=c++11 -lpthread

# 编译客户端程序
g++ -o client client.cpp -std=c++11 -lpthread
```

请务必在编译选项中包含 `-lpthread`，否则会导致链接错误（如 `undefined reference to 'pthread_create'`）。该标志用于链接 pthread 库，是多线程功能正常工作的前提。

## ▶️ 项目运行与使用方式

### 启动顺序

请严格按照以下顺序启动服务：

1. **先启动服务器**：
   ```bash
   ./server
   ```
   预期输出：
   ```
   服务器启动，监听端口: 8888
   线程池初始化完成，线程数: 64
   ```

2. **再启动客户端**：
   在另一个终端窗口执行：
   ```bash
   ./client
   ```
   预期输出：
   ```
   连接服务端成功！输入内容回车发送(exit退出)\
   请输入消息：
   ```

> **重要提示**：必须先运行服务器再启动客户端，否则客户端将因无法建立连接而报错。
